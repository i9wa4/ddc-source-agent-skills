INTRODUCTION				*ddc-source-slash-commands-introduction*

*ddc-source-slash-commands* is a ddc.vim source that provides completion for
Claude Code slash commands and skills.

When you type "/" in your buffer, it suggests available slash commands from
your commands directories and skills from your skills directories.


==============================================================================
FEATURES					*ddc-source-slash-commands-features*

- Completion only triggers when "/" is at line start or after a space
- Prevents unwanted completion in file paths (e.g., "path/to/file")
- Supports hyphenated command names (e.g., "/test-setup", "/test-run")
- Works with both matcher_head (prefix matching) and matcher_fuzzy (fuzzy
  matching)
- Supports both user-level and project-level directories
- Follows the Agent Skills specification for skills detection


==============================================================================
INSTALL					*ddc-source-slash-commands-install*

This plugin requires the following:

- Vim/Neovim with denops.vim
- ddc.vim

Using dein.vim:
>
	call dein#add('vim-denops/denops.vim')
	call dein#add('Shougo/ddc.vim')
	call dein#add('Shougo/ddc-matcher_head')  " or tani/ddc-fuzzy
	call dein#add('i9wa4/ddc-source-slash-commands')
<

Using lazy.nvim:
>
	{
	  'i9wa4/ddc-source-slash-commands',
	  dependencies = {
	    'vim-denops/denops.vim',
	    'Shougo/ddc.vim',
	    'Shougo/ddc-matcher_head',  -- or tani/ddc-fuzzy
	  },
	}
<


==============================================================================
EXAMPLES					*ddc-source-slash-commands-examples*

Basic configuration:
>
	call ddc#custom#patch_global('sources', ['slash_commands'])

	call ddc#custom#patch_global('sourceOptions', {
	  \ 'slash_commands': {
	  \   'mark': '[slash]',
	  \   'matchers': ['matcher_head'],
	  \   'minAutoCompleteLength': 1,
	  \   'isVolatile': v:true,
	  \   'forceCompletionPattern': '\/[a-zA-Z0-9_-]*',
	  \ }})
<

With custom parameters:
>
	call ddc#custom#patch_global('sourceParams', {
	  \ 'slash_commands': {
	  \   'userDirs': ['~/.claude/commands/', '~/.claude/skills/'],
	  \   'projectDirs': ['.claude/commands/', '.claude/skills/'],
	  \ }})
<

Fuzzy matching support:
>
	" Install tani/ddc-fuzzy
	call dein#add('tani/ddc-fuzzy')

	" Change matcher to fuzzy
	call ddc#custom#patch_global('sourceOptions', {
	  \ 'slash_commands': {
	  \   'matchers': ['matcher_fuzzy'],
	  \   'mark': '[slash]',
	  \   'minAutoCompleteLength': 1,
	  \   'isVolatile': v:true,
	  \   'forceCompletionPattern': '\/[a-zA-Z0-9_-]*',
	  \ }})
<

This enables patterns like "/cmd" -> "/command1".


==============================================================================
SOURCE OPTIONS			*ddc-source-slash-commands-source-options*

These are common ddc.vim source options that work well with this plugin.

				*ddc-source-slash-commands-option-mark*
mark		(string)
		Completion menu marker.

		Default: "[slash]"

				*ddc-source-slash-commands-option-matchers*
matchers	(string[])
		Matcher to use.
		- ["matcher_head"]: Prefix matching (e.g., "/my" matches
		  "/my-commit")
		- ["matcher_fuzzy"]: Fuzzy matching (e.g., "/mcom" matches
		  "/my-commit")

		For fuzzy matching, install tani/ddc-fuzzy:
		https://github.com/tani/ddc-fuzzy

		Default: ["matcher_head"]

			*ddc-source-slash-commands-option-minAutoCompleteLength*
minAutoCompleteLength	(number)
		Minimum input length to trigger completion.

		Recommended: 1

			*ddc-source-slash-commands-option-isVolatile*
isVolatile	(boolean)
		Always gather candidates on every input change.

		Recommended: v:true

		*ddc-source-slash-commands-option-forceCompletionPattern*
forceCompletionPattern	(string)
		Pattern to force completion trigger.

		Recommended: '\/[a-zA-Z0-9_-]*'


==============================================================================
PARAMS					*ddc-source-slash-commands-params*

				*ddc-source-slash-commands-param-userDirs*
userDirs	(string[])
		User-level directories to scan for commands and skills.
		Paths starting with "~" are expanded to the home directory.

		Default: ["~/.claude/commands/", "~/.claude/skills/"]

				*ddc-source-slash-commands-param-projectDirs*
projectDirs	(string[])
		Project-level directories to scan for commands and skills.
		Relative paths are resolved from the current working directory.

		Default: [".claude/commands/", ".claude/skills/"]


==============================================================================
SKILLS					*ddc-source-slash-commands-skills*

This plugin follows the Agent Skills specification (https://agentskills.io).

For skills directories, only subdirectories containing a SKILL.md file are
recognized as valid skills.

NOTE: Skills detection only works for directories named "skills". Custom
directory names are treated as command directories.
>
	~/.claude/skills/
	  my-skill/
	    SKILL.md      " Required - makes this a valid skill
	    other.md      " Optional additional files
	  incomplete/
	    readme.md     " Not recognized (no SKILL.md)
<

Completion items include menu labels showing the source.

For default configuration:

- [commands:user]    - User-level command
- [commands:project] - Project-level command
- [skills:user]      - User-level skill
- [skills:project]   - Project-level skill

NOTE: Menu labels use the directory name. Custom directories will show
labels like [my-dir:user] based on their basename.


==============================================================================
MIGRATION				*ddc-source-slash-commands-migration*

If upgrading from v1.x, the following parameters have been removed:

- commandsDir: Use |ddc-source-slash-commands-param-userDirs| instead
- extensions: Removed (hardcoded to ".md")

Migration example:
>
	" Before (v1.x)
	call ddc#custom#patch_global('sourceParams', {
	  \ 'slash_commands': {
	  \   'commandsDir': '~/my/commands',
	  \ }})

	" After (v2.x)
	call ddc#custom#patch_global('sourceParams', {
	  \ 'slash_commands': {
	  \   'userDirs': ['~/my/commands/'],
	  \ }})
<
